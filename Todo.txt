                1. Fetcher issue, The current HTTPFetcher uses a simple HTTP GET request which only retrieves the initial HTML. For Single Page Applications (SPAs), JavaScript needs to execute to render the full content. We need to use a headless browser approach.
                    Solution
                    Create a new BrowserFetcher that uses the chromedp package to launch a headless Chrome browser, wait for the page to fully load (including JS execution), then read the rendered HTML body.
                    Keep the existing HTTPFetcher for non-SPA pages.
                    https://github.com/chromedp/chromedp
